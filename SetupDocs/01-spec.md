# 01. 仕様（何が起きるか）

実装の実挙動に基づく仕様です。

## コマンド

- `/announce`（引数なし）。Discord 上で実行すると入力ダイアログ（Modal）が開く。

## Modal（入力ダイアログ）

| 入力欄 | 形式 | 必須 | 説明 |
| --- | --- | --- | --- |
| 告知本文 | 複数行 | 必須 | Slack に投稿する本文 |
| 確認 (SEND と入力すると送信) | 1行 | 必須 | `SEND` と入力したときだけ送信する安全弁 |

## 実行の流れと分岐

1. **署名検証**: Discord からのリクエストを Ed25519 で検証。不正なら `401`。
2. **PING**: Discord の Endpoint 検証（PING）には PONG を返す。
3. **権限チェック**: 実行者が許可ユーザー（`ALLOWED_DISCORD_USER_IDS`）でなければ、
   「このコマンドを実行する権限がありません。」と本人にのみ表示して中断。
   - コマンド実行時と Modal 送信時の両方でチェックする。
4. **確認欄チェック**: 確認欄が `SEND` 以外なら
   「確認欄に SEND と入力されていないため、送信を中断しました。」で中断。
   - `SEND` は**前後空白を除いた完全一致**（大文字小文字を区別）。
5. **本文チェック**: 告知本文が空なら「告知本文が空のため中断しました。」で中断。
6. **送信処理**（ここまで通過したときだけ）:
   - まず Discord に「処理中」の遅延応答（deferred）を返す（3秒制限対策）。
   - バックグラウンドで CSV 取得 → メッセージ生成 → Slack 投稿。
   - 完了後、最初の応答メッセージを結果で更新する。

## Slack への投稿フォーマット

チャンネルごとに1メッセージ。1行目に対象者のメンション、空行、本文。

```
<@U012AAAA> <@U012BBBB>

（告知本文）
```

- メンションは半角スペース区切り。
- 本文は前後の空白のみ除去（途中の改行は保持）。

## 結果表示（Discord・本人のみ）

- 通常: `投稿完了: 成功 N件 / 失敗 M件`
- 失敗があれば、続けて失敗チャンネルと理由を一覧表示。
- 対象者が0人: `投稿対象の参加者が見つかりませんでした。`
- 例外発生時: `エラーが発生しました: <理由>`
- いずれも Discord の本文上限（約2000字）を超える場合は自動で切り詰める。

## やらないこと（スコープ外）

- 予約投稿・宛先の手動選択・テンプレート管理などはなし。
- 投稿ログの保存（DB）なし。結果は Discord の応答メッセージのみ。
