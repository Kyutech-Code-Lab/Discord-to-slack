# AGENT.md

## 目的

このリポジトリは、DiscordからSlackへ告知を送るための最小ツールです。

運営者がDiscordで `/announce` を実行すると入力ダイアログが開き、入力された本文をSlackの各グループチャンネルへ個人メンション付きで投稿します。

## 作業前に読むもの

まずこの `AGENT.md` を読んでください。

その後、作業内容に応じて `docs/` 配下の必要なドキュメントを読んでください。

* 仕様確認が必要な場合: `docs/SPEC.md`
* 環境構築・デプロイ・動作確認が必要な場合: `docs/SETUP.md`

仕様で迷った場合は `docs/SPEC.md` を正としてください。

## 基本方針

* 実装は小さく保つ
* 既存の設計に従う
* 不要な機能を追加しない
* 仕様にない判断を勝手にしない
* わからないことがあれば、作業を進める前にユーザーへ確認する
* 複数の実装方針があり、どれを選ぶべきか判断できない場合はユーザーに確認する
* 明確な指示がない限り、破壊的な変更は行わない

## 禁止事項

以下は禁止です。

* 勝手に `git push` しない
* 勝手に本番デプロイしない
* 勝手に環境変数を変更しない
* 勝手にDBや外部サービスを追加しない
* 勝手に認証方式を変更しない
* 勝手にUIや操作フローを変更しない
* 勝手に依存ライブラリを大量追加しない
* 秘密情報をログに出さない
* `.env` やトークンをコミットしない
* `docs/SPEC.md` にない機能を追加しない

## Git運用

* 作業前に現在の差分を確認する
* ユーザーの未コミット変更を上書きしない
* 変更内容が大きくなる場合は、作業単位を分ける
* コミットはユーザーに指示された場合のみ行う
* pushは絶対にユーザーの明示指示がある場合のみ行う

## 実装ルール

使用するもの：

* Next.js App Router
* TypeScript
* Vercel Functions
* Discord Interactions Endpoint
* Slack Web API
* Google Sheets CSV

使用しないもの：

* 常駐型Discord Bot
* discord.py
* discord.jsのGateway Client
* FastAPI
* DB
* 管理画面
* キューサービス

実装時のルール：

* Discord、Slack、Google Sheets、告知生成の処理を分離する
* 可能な限り標準の `fetch` を使う
* 型を明確に書く
* エラーを握りつぶさない
* Discordリクエストの署名検証を必ず行う
* 許可されたDiscordユーザーだけが実行できるようにする
* Slack投稿前に、入力値と参加者情報を検証する

## ドキュメントの読み分け

以下を確認したいときに読む。

* `/announce` の挙動
* Discord Modalの入力項目
* Slackへの投稿形式

以下を確認したいときに読む。

* Discord Appの作成
* Interactions Endpoint URLの設定
* Slash Commandの登録
* Slack Appの作成
* Google Sheets CSVの準備
* Vercelの環境変数設定
* デプロイ手順
* 動作確認手順

## 不明点がある場合

以下の場合は、推測で進めずユーザーに確認する。

* 仕様に書かれていない挙動が必要になった場合
* 既存仕様と矛盾する指示がある場合
* セキュリティに関わる判断が必要な場合
* 外部サービスや追加ライブラリの導入が必要な場合
* データ構造を変更する必要がある場合
* 本番環境に影響する可能性がある場合

## 完了条件

以下を満たしたら実装完了です。

* `pnpm build` が通る
* DiscordのEndpoint検証が通る
* `/announce` で入力ダイアログが開く
* 入力後、Slackへ投稿される
* 成功/失敗の結果がDiscordに返る
* 変更内容を簡潔に説明できる
